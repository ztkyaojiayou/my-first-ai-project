# 用户管理系统 - 项目说明

## 📌 项目概述
基于前后端分离架构的用户管理系统演示项目，包含用户注册、登录功能。

## 🏗️ 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                        浏览器                                │
│                   (http://localhost)                         │
└──────────────────────────────┬──────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────┐
│                  nginx (Docker 容器)                        │
│                    端口: 80                                 │
├─────────────────────────────────────────────────────────────┤
│  /            → 托管前端静态文件 (index.html)                │
│  /api/*       → 反向代理到 Spring Boot 后端                 │
└──────────────────────────────┬──────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────┐
│              Spring Boot 后端服务                            │
│                    端口: 8080                               │
├─────────────────────────────────────────────────────────────┤
│  Controller → Service → InMemoryStorage (内存存储)           │
│                      ↓                                       │
│                    JWT + Redis                              │
└─────────────────────────────────────────────────────────────┘
```

## ✅ 已实现功能

### 1. 后端功能
| 功能 | 接口 | 说明 |
|------|------|------|
| 用户注册 | `POST /api/user/register` | 用户名密码注册 |
| 用户登录 | `POST /api/user/login` | 登录并返回 JWT Token |
| 获取用户信息 | `GET /api/user/info` | 获取当前登录用户信息 |
| 退出登录 | `POST /api/user/logout` | 登出并加入 Token 黑名单 |
| 健康检查 | `GET /api/health/check` | 系统状态检查 |

### 2. 前端功能
| 页面 | 功能 |
|------|------|
| 登录页 | 用户名密码登录 |
| 注册页 | 新用户注册 |
| Tab 切换 | 登录/注册页面切换 |

## 🛠️ 技术栈

### 后端
- Spring Boot 2.7.2
- JWT (Token 认证)
- Redis (Token 黑名单)
- InMemoryStorage (内存数据存储)

### 前端
- 原生 JavaScript (无框架)
- 原生 CSS (无 UI 库)
- HTML5

### 部署
- Docker + nginx (前端托管 + 反向代理)
- Maven (Spring Boot 构建)

## 🚀 访问地址

### 通过 nginx 访问（需要启动 nginx）
- 前端首页: http://localhost
- 健康检查: http://localhost/health
- 后端 API: http://localhost/api/*

### 直接访问后端（仅启动 Spring Boot 即可）
- 后端服务端口: **8080**
- 健康检查: http://localhost:8080/api/health/check
- 用户登录: http://localhost:8080/api/user/login
- 用户注册: http://localhost:8080/api/user/register
- 获取用户信息: http://localhost:8080/api/user/info
- 退出登录: http://localhost:8080/api/user/logout

### API 测试示例

```bash
# 健康检查
curl http://localhost:8080/api/health/check

# 用户登录
curl -X POST http://localhost:8080/api/user/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}'

# 用户注册
curl -X POST http://localhost:8080/api/user/register \
  -H "Content-Type: application/json" \
  -d '{"username":"newuser","password":"123456","nickname":"新用户"}'

# 获取用户信息（需携带 Token）
curl http://localhost:8080/api/user/info \
  -H "Authorization: Bearer <your_token>"
```

## 🔧 运维命令

### 启动服务

#### 1. 启动后端（必选）
```bash
cd trae-demo
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.8.0_181.jdk/Contents/Home  # 可选，指定 JDK 8
mvn spring-boot:run
```

#### 2. 启动 nginx（可选，仅需要完整前端时启动）
```bash
docker run -d --name nginx \
  -p 80:80 \
  -v $(pwd)/nginx/html:/usr/share/nginx/html:ro \
  -v $(pwd)/nginx/conf/nginx.conf:/etc/nginx/nginx.conf:ro \
  --add-host host.docker.internal:host-gateway \
  nginx:alpine
```

### 停止服务

#### 1. 停止 nginx
```bash
docker stop nginx
docker rm nginx
```

#### 2. 停止 Spring Boot
```bash
ps aux | grep java | grep -E "(spring-boot|userdemo)" | grep -v grep | awk '{print $2}' | xargs kill -9
```

#### 3. 一键停止所有服务
```bash
docker stop nginx 2>/dev/null; docker rm nginx 2>/dev/null; ps aux | grep java | grep -E "(spring-boot|userdemo)" | grep -v grep | awk '{print $2}' | xargs kill -9 2>/dev/null
```

### 查看服务状态

#### 查看 nginx
```bash
docker ps
docker logs nginx
```

#### 测试后端 API（直接访问后端 8080）
```bash
curl http://localhost:8080/api/health/check
```

## 💡 注意事项

1. 数据保存在内存中，服务重启后会丢失
2. 本地未启动 Redis 的话，Token 黑名单功能可能不可用，但不影响登录注册
3. Java 版本要求: JDK 8
4. 仅测试后端 API 时只需启动 Spring Boot（端口 8080），无需启动 nginx
5. 强制推送 (`git push --force`) 会覆盖远程历史，务必确认操作后再使用
6. 使用 `rm -rf` 删除目录前务必确认路径，避免误删重要文件
