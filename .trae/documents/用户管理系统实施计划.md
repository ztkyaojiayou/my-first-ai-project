# 用户管理系统实施计划

## 一、项目技术栈

| 组件 | 版本 |
|------|------|
| Spring Boot | 2.7.2 |
| MyBatis-Plus | 3.5.1 |
| MySQL | 8.0 |
| Redis | 6.x/7.x |
| Java | 17 |

## 二、项目架构

```
├── controller          # 控制层（处理请求）
├── service            # 业务层（逻辑处理）
├── mapper             # 数据访问层（MyBatis-Plus）
├── entity             # 实体类
├── dto                # 数据传输对象
├── config             # 配置类
├── common             # 通用类（常量、响应封装等）
└── util               # 工具类
```

## 三、核心功能模块

### 1. 用户模块
- 用户注册
- 用户登录（JWT认证）
- 用户信息查询
- 用户信息修改
- 用户删除
- 密码加密存储（BCrypt）

### 2. 角色权限模块
- 角色管理（CRUD）
- 权限管理（CRUD）
- 用户角色分配
- 角色权限分配
- 基于Redis的权限缓存

### 3. 系统工具模块
- Redis缓存工具类
- JWT工具类
- 统一响应封装

## 四、数据库设计

### 1. 用户表（sys_user）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| username | varchar(50) | 用户名 |
| password | varchar(100) | 密码（加密） |
| nickname | varchar(50) | 昵称 |
| email | varchar(100) | 邮箱 |
| phone | varchar(20) | 手机号 |
| avatar | varchar(255) | 头像 |
| status | tinyint | 状态（0禁用/1启用） |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |

### 2. 角色表（sys_role）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| name | varchar(50) | 角色名 |
| code | varchar(50) | 角色代码 |
| description | varchar(255) | 描述 |
| status | tinyint | 状态 |
| create_time | datetime | 创建时间 |

### 3. 权限表（sys_permission）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| name | varchar(50) | 权限名 |
| code | varchar(50) | 权限代码 |
| type | tinyint | 类型（1菜单/2按钮） |
| url | varchar(255) | 请求路径 |
| pid | bigint | 父级ID |
| create_time | datetime | 创建时间 |

### 4. 用户角色表（sys_user_role）
| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | bigint | 用户ID |
| role_id | bigint | 角色ID |

### 5. 角色权限表（sys_role_permission）
| 字段 | 类型 | 说明 |
|------|------|------|
| role_id | bigint | 角色ID |
| permission_id | bigint | 权限ID |

## 五、实施步骤

### 阶段一：项目初始化
1. 创建Spring Boot项目，引入依赖
2. 配置application.yml（数据库、Redis、JWT等）
3. 集成MyBatis-Plus，配置分页插件
4. 集成Redis，配置序列化方式

### 阶段二：基础功能实现
1. 设计并创建数据库表
2. 生成实体类（Entity）
3. 实现通用BaseMapper
4. 实现CRUD基础接口

### 阶段三：认证授权实现
1. 实现JWT工具类
2. 实现登录接口（密码校验+Token生成）
3. 配置Spring Security
4. 实现权限校验拦截器
5. Redis缓存用户权限信息

### 阶段四：业务功能完善
1. 实现角色管理接口
2. 实现权限管理接口
3. 实现用户角色分配接口
4. 实现基于Redis的权限缓存更新机制

### 阶段五：工具类与配置
1. 统一响应封装（Result类）
2. 全局异常处理
3. Redis工具类封装
4. 跨域配置

## 六、关键配置说明

### Maven依赖
- spring-boot-starter-web
- mybatis-plus-boot-starter
- mysql-connector-java
- spring-boot-starter-data-redis
- jjwt（JWT支持）
- spring-boot-starter-security
- hutool（工具库）

### Redis缓存策略
- 用户信息缓存：key=user:info:{userId}，TTL=30分钟
- 权限信息缓存：key=permission:{userId}，TTL=1小时
- Token黑名单：key=token:blacklist:{token}，TTL=Token剩余有效期

## 七、API接口设计

### 用户接口
- POST /api/user/register - 用户注册
- POST /api/user/login - 用户登录
- GET /api/user/info - 获取当前用户信息
- PUT /api/user - 修改用户信息
- DELETE /api/user/{id} - 删除用户

### 角色接口
- GET /api/role/list - 角色列表
- POST /api/role - 新增角色
- PUT /api/role - 修改角色
- DELETE /api/role/{id} - 删除角色

### 权限接口
- GET /api/permission/list - 权限列表
- POST /api/permission - 新增权限
- PUT /api/permission - 修改权限
- DELETE /api/permission/{id} - 删除权限

### 用户角色接口
- GET /api/user/{userId}/roles - 获取用户角色
- PUT /api/user/{userId}/roles - 分配用户角色
